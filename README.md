## 🧠 How I Work

- 요구사항을 **기능 단위가 아닌 도메인 단위**로 해석합니다.
- “일단 구현”보다 **왜 이렇게 설계해야 하는지**를 먼저 고민합니다.
- 성능 이슈, 예외 상황, 확장 가능성을 코드 수준에서 명확히 드러내려 합니다.
- 읽기 쉬운 코드와 명확한 책임 분리를 중요하게 생각합니다.
